## What's been DONE in this session (Phase 1 work, partial)

1. **Popover/Dialog/Sheet close-animation bug — partial fix (safety net only, shipped):**
   - Root cause diagnosed: reka-ui's `Presence`/`usePresence` state machine can get permanently stuck in `unmountSuspended` on interrupted/rapid-toggle animations, leaving closed overlays fully visible and interactive.
   - **Applied**: added `data-[state=closed]:pointer-events-none` to the class strings in `resources/js/components/ui/popover/PopoverContent.vue`, `resources/js/components/ui/dialog/DialogContent.vue`, `resources/js/components/ui/sheet/SheetContent.vue`. This stops the worst symptom (a stuck overlay blocking clicks on real UI underneath it).
   - Also removed a redundant `transition ease-in-out` Tailwind utility from `SheetContent.vue` (line ~99) that was stacking with the `animate-in`/`animate-out` keyframe classes and likely causing the Sheet's specific "stuck off-screen on first open" symptom — kept the `duration-300`/`duration-500` utilities, which `tw-animate-css` respects directly.
   - **NOT done / deferred**: the deeper root-cause fix (convert to reka-ui's `forceMount` + native Vue `<Transition>` pattern) — turned out to need injecting reka-ui's internal root context (`injectPopoverRootContext()` / dialog equivalent) into the wrapper components, which isn't currently exposed to them. This is nontrivial surgery on a primitive used on nearly every page in the app. Deferred rather than rushed, given a concurrent session was actively editing overlapping areas of the codebase at the time. **Next step if resuming**: check whether `reka-ui`'s public API exports `injectPopoverRootContext`/`injectDialogRootContext` (or equivalent) for consumer use — if not, this fix needs a different approach (e.g. reading `data-state` via a ref + `MutationObserver`, or accepting the CSS safety net as the permanent fix and closing this out).
   - **Considered and explicitly reverted**: fixing dead `[data-radix-dialog-*]` CSS selectors in `resources/css/app.css` (~line 1306) to the real `[data-slot=...]` ones reka-ui actually emits. Decided against reviving this — the `!important`-heavy rules (blur backdrop, forced card background, gradient destructive buttons) have been dead code for a while and reviving them would be an untested, app-wide visual change disproportionate to a "cleanup" item. Left as-is (still dead/no-op).
